# Contributing Guidelines

Thank you for your interest in contributing to this fraud detection resources project!

## How to contribute

### Adding papers
- Add PDF to appropriate folder in `papers/`
- Update relevant documentation with summary
- Include practical applicability notes

### Adding code examples
- Follow existing code structure in `code/examples/`
- Include comprehensive docstrings
- Add requirements if new dependencies needed
- Test with sample datasets

### Adding datasets
- Add preprocessing scripts to `datasets/preprocessing_scripts/`
- Document dataset characteristics
- Include usage examples

### Reporting bugs
- Use GitHub issues
- Include reproduction steps
- Provide environment details

### Suggesting features
- Use GitHub discussions for ideas
- Explain use case and benefits

## Pull request process
1. Fork the repository
2. Create feature branch
3. Make changes with documentation
4. Submit PR with clear description

## Questions?
Open a discussion or contact the maintainers!
